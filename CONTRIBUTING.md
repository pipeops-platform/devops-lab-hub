# Contributing Guide

Thank you for contributing to the DevOps Hybrid Lab documentation and standards repository.

## Branching Model

- Use short-lived feature branches from `main`
- Open pull requests for all changes
- Keep updates focused and atomic

## Commit Messages

Use clear, action-oriented commits.

Suggested prefixes:

- `docs:` documentation updates
- `feat:` new project standard/runbook
- `fix:` corrections
- `chore:` maintenance

## Pull Request Expectations

- Describe rationale and affected project phase/repository
- Keep language in English
- Ensure links and references are valid

## Quality Checks

Before opening a PR:

1. Verify document structure consistency
2. Verify link integrity in updated sections
3. Ensure terminology consistency across repositories

## Documentation Rules

- English only
- Keep runbooks actionable and concise
- Keep phase and repository references aligned
